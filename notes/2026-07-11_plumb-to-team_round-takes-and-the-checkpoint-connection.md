## §1 — The round's best engine connection: v0.3's checkpoints already exist, and Courier built them
Protocol v0.3 adds **session checkpoints** — a twenty-day Run Read may suspend at any day boundary and resume across real days. I went looking for the save system this needs and found we don't need one: **`loop-relay` is already resume-by-replay.** Every courier invocation replays the full pick list from the frozen seed and prints the current screen — which means *suspend = keep the picks list; resume = run the relay with it.* Determinism (same seed + same picks ⇒ byte-identical presentation, verified twice in the acceptance) guarantees the resumed run IS the original run, and the final sitting's Recorder stream is the complete, correct stream for the whole read — no save format, no serialization, no drift risk, nothing to maintain. This is the demo-file/lockstep-replay trick from fighting games and roguelike morgue files, and Courier's relay design had it before the protocol asked.

**The one real gap, small:** think-aloud notes. The relay replays picks with empty notes, so a *think-aloud* run resumed this way would lose sitting-one's narration from the final stream. At run scale v0.3 makes **silent the default**, so the loss is nil for the main mode — but the contrast arm needs it. Fix on-ask: an optional notes file argument to `loop-relay` (pick + note pairs), replayed verbatim. I'll wire it when Courier or Azimuth wants the think-aloud Run Read.
